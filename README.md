# Método Tesis 60-40 IA™ 🎓

Bienvenido al repositorio principal del **Método Tesis 60-40 IA™**, un sistema avanzado de asistencia ética para la investigación académica.

## 📖 Contexto: ¿Qué es el Método 60-40?

El **Método Tesis 60-40 IA™** es un sistema integral (basado en Inteligencia Artificial) que guía al tesista desde la concepción de la idea hasta la sustentación final. Su filosofía central se define en una división clara de responsabilidades:

- **60% - El Tesista:** Aporta el pensamiento crítico, los datos de la vida real, el juicio metodológico, la voz propia, la interpretación y las decisiones finales.
- **40% - La IA (Sistema):** Aporta la estructura, corrección de estilo, síntesis guiada, verificación de formato (APA 7), detección de inconsistencias y la organización lógica del documento.

> **Regla de Oro:** La IA asiste, pero el tesista *decide*. Ningún resultado generado por el sistema reemplaza el criterio y responsabilidad del investigador.

---

## 🧭 Instructivo y Fases de Trabajo

El sistema está diseñado para utilizarse de manera secuencial a través de 3 grandes fases. 

### Fase 1: Planificación (El Cimiento)
Esta fase define la ruta de la investigación.
- Comienza con el comando `/diagnostico` para evaluar el estado actual del proyecto.
- Utiliza `/planificar [titulo | problema | objetivos | variables | metodologia | consistencia]` para ir construyendo, paso a paso, los elementos fundamentales del Capítulo I.

### Fase 2: Desarrollo (La Investigación)
En esta fase se redacta el cuerpo de la tesis, basado en literatura real y datos verídicos aportados por el tesista.
- Usa `/literatura` para estructurar los antecedentes y bases teóricas.
- Emplea `/escribir [sección]` para redactar el borrador del marco teórico, metodología, resultados, discusión, etc. *(Nota: La sección de resultados requiere obligatoriamente que el tesista proporcione los datos recolectados).*
- Utiliza `/analizar [cuantitativo | cualitativo | mixto]` para obtener una guía algorítmica sobre qué tipo de pruebas y análisis estadísticos/cualitativos aplicar.

### Fase 3: Refinamiento (La Calidad)
Fase dedicada a la corrección, validación y preparación final.
- Ejecuta `/revisar [todo | coherencia | apa]` para que los "agentes críticos" auditen la tesis.
- Aplica `/corregir [estilo | apa]` para mejorar la redacción y formato final.
- Utiliza `/sustentar [simulacro | estructura | diapositivas]` para prepararte para la defensa ante el jurado usando la técnica **R-E-C** (Reconocer + Explicar + Conectar).

## 🛠️ Reglas Éticas y Gobernanza (Inmutables)

1. **Cero Inventos:** El sistema *NUNCA* inventa citas bibliográficas, autores, teorías ni estadísticas. 
2. **Validación Obligatoria:** Las fuentes no comprobadas se marcan siempre como `[PENDIENTE DE VERIFICACIÓN]`.
3. **Decisión Humana:** El sistema nunca reemplaza el juicio metodológico del tesista ni garantiza la aprobación académica per se.
4. **Verificación Continua:** Todos los borradores pasan por un proceso de crítica antes de ser entregados, y las decisiones del tesista siempre prevalecen.

## 📂 Estructura del Repositorio

- `/tesis/`: Directorio principal donde se construirá el documento final y sus anexos.
- `/templates/`: Plantillas paso a paso (Diagnóstico, Problema, Marco Teórico, etc.).
- `/quality_reports/`: Planes de sesión y auditorías de coherencia (el registro de la calidad).
- `CLAUDE.md`: La "constitución" del sistema, con directrices internas y configuraciones del comportamiento de la IA.
- `MANUAL_REFERENCIA.md`: Referencia rápida de todos los comandos (`skills`) y reglas del sistema.
- `MANUAL_CLIENTES.html`: Documentación visual e interactiva para el usuario final.
